Configuração da base de dados

Versão simples com CSV

Coloca os alunos em alunos.csv com as colunas aluno_id,nome,turma,curso. Em GitHub Pages, Netlify, Vercel ou Cloudflare Pages o ficheiro pode ser lido diretamente pelo navegador.

Google Sheets

Publica a folha como CSV e substitui, em app.js, a constante DATA_SOURCE pelo URL publicado. Evita dados sensíveis.

Alternativa Supabase

Cria as tabelas alunos e escolhas, ativa autenticação se necessário e troca apenas a camada studentRepository e choiceRepository.

Administração

Exportar resultados

As escolhas desta versão ficam guardadas no navegador, em localStorage. Para recolha centralizada, usa Google Forms/Sheets ou Supabase.

Publicar no GitHub Pages

  1. Cria um repositório no GitHub e envia a pasta escolhas-12ano.
  2. Nas definições do repositório, abre Pages.
  3. Escolhe a branch principal e a pasta raiz do projeto.
  4. Confirma que index.html, style.css, app.js e alunos.csv ficam publicados juntos.

Aplicação escolar demonstrativa. Recolhe apenas os dados mínimos necessários.
